# Planwerk

**Human-in-the-loop development workflow — AI proposes, you decide**

---

*AI-assisted planning · Test-driven development · Quality guard rails · Human-guided execution*

</div>

## Overview

Planwerk bridges the gap between **"what should we build?"** and **"it's merged."** You define the features you need — AI elaborates your ideas, works out the details, and delivers concrete proposals for user stories, tasks, and architecture. Every implementation follows **strict test-driven development** — tests first, code second, no exceptions. Automated test suites, CI pipelines, and multi-agent code reviews form **layered quality guard rails** that catch problems early. The AI remembers your project's patterns and conventions across sessions. But **every decision stays with you** — you review, steer, and approve. Planwerk automates the work, not the control.

## How it works

| Phase | Description |
|-------|-------------|
| <nobr>:mag: **Discover**</nobr> | You describe what you need — AI analyzes your codebase and elaborates your ideas into actionable, concrete proposals |
| <nobr>:world_map: **Plan**</nobr> | Breaks features into user stories, tasks, acceptance criteria, and test specifications using SPARC methodology |
| <nobr>:hammer_and_wrench: **Build**</nobr> | Every line of code starts with a failing test — strict Red-Green-Refactor TDD enforces that nothing ships untested |
| <nobr>:white_check_mark: **Review**</nobr> | AI performs multi-layered code review — a dedicated devil's advocate agent challenges every change before you give final approval |
| <nobr>:rocket: **Ship**</nobr> | Creates PRs, monitors CI checks, and addresses review comments — nothing merges without your sign-off |

## Key Features

| Feature | Description |
|---------|-------------|
| <nobr>:mag_right: **AI-Driven Elaboration**</nobr> | Takes your feature ideas and uses Claude to work out the details — architecture, dependencies, and concrete implementation proposals |
| <nobr>:clipboard: **Structured Planning**</nobr> | Features decomposed into user stories, tasks, acceptance criteria, formal requirements, and explicit scope boundaries |
| <nobr>:test_tube: **Test-Driven Development**</nobr> | Test specifications defined during planning — tests are written first, always — Red-Green-Refactor with automated phase tracking ensures no shortcuts |
| <nobr>:busts_in_silhouette: **Multi-Agent Review**</nobr> | Dedicated review and devil's advocate agents scrutinize every change — catching issues before they reach your desk |
| <nobr>:octocat: **GitHub Integration**</nobr> | PR creation, CI check monitoring, review handling, and merge management — CI must pass before anything proceeds |
| <nobr>:shield: **Quality Guard Rails**</nobr> | Automated test suites, linting, type checks, and CI pipelines form multiple layers of defense — broken builds block progress |
| <nobr>:chart_with_upwards_trend: **Web Dashboard**</nobr> | Real-time Next.js frontend for feature tracking and terminal output |
| <nobr>:arrows_counterclockwise: **Batch Execution**</nobr> | Run multiple features in sequence with dependency handling |
| <nobr>:deciduous_tree: **Git Worktree Support**</nobr> | Isolated branches per feature for parallel development |
| <nobr>:bookmark_tabs: **Plan Extraction**</nobr> | Import features from existing PLAN.md or ROADMAP.md files |
| <nobr>:brain: **Pattern Memory**</nobr> | Learns and remembers project conventions, code patterns, and architectural decisions across sessions |
| <nobr>:receipt: **Prompt Traceability**</nobr> | Tracks AI usage per feature for cost and audit visibility |
